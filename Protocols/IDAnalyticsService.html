IDAnalyticsService Protocol Reference

Conforms to NSObject
Declared in IDAnalyticsService.h

Overview

Provides a consistent interface for adaptation to support multiple analytics services. Classes conforming to this protocol can be registered with the IDAnalyticsMediator to receive all events and other method calls.

trackingEnabled

Enables or disables tracking to the analytics API.

@property (nonatomic, assign, getter=isTrackingEnabled) BOOL trackingEnabled

Discussion

When disabled, events cannot be recorded. Ensure that events are neither submitted to the API nor recorded in a local cache.

The default value is YES; if tracking is disabled be sure to update this property immediately after initializing the service.

Instance Methods

registerRemoteNotificationsDeviceToken:

Registers the given device to receive push notifications from services that support push notification engagement.

- (void)registerRemoteNotificationsDeviceToken:(NSData *)deviceToken

Parameters

deviceToken

The token provided to the app delegate in -application:didRegisterForRemoteNotificationsWithDeviceToken:

Discussion

For services that support push notifications, the device token will be associated with the user such that notifications can be sent to any of their devices. The device token must be as-received from -application:didRegisterForRemoteNotificationsWithDeviceToken: after registering for remote notifications.

reset

Resets any user state that is cached on the device.

Discussion

This is useful when a user logs out and you want to clear the identity, or for apps that include the option to fully disable tracking.
